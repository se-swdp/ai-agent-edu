Tips 01-20 Advanced Commands Files Prompts
Tips
01코딩 도구가 아니다
다목적 인터페이스. 영상 편집(ffmpeg), 용량 정리, CSV 분석, 문서 작성 — 컴퓨터에서 하는 디지털 작업을 자연어로 위임할 수 있다.
023시간 회의의 법칙
Context Window = 화이트보드. 대화 3시간 넘으면 성능 하락. 새 주제엔 새 대화. 이전 맥락이 아까우면 /compact 또는 HANDOFF.md.
03빠르게 만들자
완벽주의는 AI 시대 가장 비싼 사치. 30분 계획하고 프로토타입부터. 뼈대는 30분이면 나온다 — 부족하면 고치면 그만.
04Git 3줄 요약
Commit = 저장, PR = 검토 요청, Push = 외부 공개. 명령어 외울 필요 없다 — "커밋해줘"라고 말하면 끝.
05계단식 작업 방식
터미널 탭 여러 개 = 동시 병렬 작업. 각 탭의 Claude가 독립적으로 작업. 프론트/백엔드/문서/리서치를 동시에.
06Git Worktree
여러 기능을 충돌 없이 동시 개발. "작업 트리 만들어줘"라고 말하면 git worktree add 자동 실행.
07AI가 AI에게 심부름
Claude Code → Gemini → WebSearch → 종합 보고서. 인간은 구경만. Claude가 다른 AI에게 웹 페이지 읽기를 시키고 결과를 종합.
08매뉴얼 3단계
CLAUDE.md (기본 매뉴얼) → Skills (업무별 특화) → Plugins (스킬 묶음). 나만의 업무 매뉴얼을 Claude에게 줄 수 있다.
09슬래시 명령어
/compact 압축, /clear 새 대화, /usage 사용량, /copy 복사, /chrome 브라우저, /commit 커밋.
10터미널이 감옥이 아니다
클립보드 복사, 브라우저 연동, HTML/PDF 생성 후 바로 열기, VS Code에서 직접 편집 — 출력의 자유.
11바이브 코딩의 진짜 의미
나한테 딱 맞는 도구를 직접 만드는 것. 범용 SaaS 시대에서 → 나만의 맞춤 도구를 AI가 만들어주는 시대.
12음성으로 코딩하기
SuperWhisper — 로컬 음성 인식. 타이핑보다 3배 빠른 입력. 오타/문법 틀려도 Claude가 의도를 파악.
13테스트 먼저 쓰기
TDD + Claude Code = 자율 개발 루프. "테스트를 먼저 작성하고, 통과할 때까지 구현해줘" — Claude가 알아서 반복.
14배운 건 나누자
팀 내 공유가 가장 빠른 학습. 삽질 경험, 꿀팁, 스킬 공유. 커뮤니티 블로그, 사내 위키 — 한 사람의 팁이 팀 전체 생산성을 높인다.
15클로드 코드에게 물어보세요
모르면 직접 물어보면 된다. "너 뭘 할 수 있어?", "이 에러 해결해줘" — 완벽한 프롬프트를 쓸 필요 없다.
16프롬프트 철학
"먼저 조사해줘" — 실행 전 리서치가 핵심. 조사 → 분석 → 실행 순서를 지키면 결과물 품질이 2배.
17결과 검증
신뢰하되 검증. 1차 결과물 → 다른 AI에게 "이거 검증해줘" → 차이점 기반 최종 판단. 테스트 코드로 자동 검증도 가능.
18맥락 주입
AI에게 역할과 목적을 명시하면 충실도가 극적으로 올라간다. "나는 보안 엔지니어이고 이건 보안 감사다" → 상세 분석 제공.
19백엔드 모델 결합
Claude Code(코드/실행/자동화) + Gemini 3 Pro(1M 토큰/웹/멀티모달) = 최강 조합. 긴 문서는 Gemini에게, 실행은 Claude에게.
20Harness 내재화
남의 SaaS보다 내 것을 만들자. 직접 만든 Skill + CLAUDE.md = 완전한 제어. 만드는 과정 자체가 AI 활용 가속의 시작점.
Advanced
•CLAUDE.md 실전 작성법
프로젝트의 DNA를 Claude에게 알려주기
# CLAUDE.md

## 프로젝트 개요
- 데이터 분석 도구
- Python 3.11+ / pandas / matplotlib

## 코딩 규칙
- 변수명: snake_case (영어)
- 주석: 한국어 OK
- 타입 힌트: 모든 함수에 필수

## 커밋 규칙
- feat: / fix: / docs: / refactor:
•나만의 Skill 만들기
반복 업무를 한 줄 명령어로 자동화
# ~/.claude/skills/meeting-report.md
---
name: meeting-report
description: 회의록을 임원 보고서로 변환
trigger: /meeting-report
---

입력된 회의록을 다음 형식으로 변환하세요:
1. 핵심 결정사항 (3줄 이내)
2. Action Items (담당자, 기한 포함)
3. 리스크 & 이슈 (영향도 표시)
4. 다음 회의 안건
•Agentic Workflow
멀티 스텝 자동화 — Claude Code를 자율 에이전트로 활용
// 프롬프트 예시
"다음 작업을 순서대로 수행해줘:
1. competitor_news/ 폴더의 기사를 모두 분석해
2. 주요 트렌드 3가지를 추출해
3. 각 트렌드별 시사점을 정리해
4. 임원 보고서(HTML)로 만들어줘"
•멀티 에이전트 패턴
Terminal 여러 개를 동시에 운영. 메인 개발 / 테스트 & 디버그 / 문서 & 리서치 — 인간은 오케스트라 지휘자.
•프롬프트 패턴
Bad: "코드 짜줘"  /  Good: "users 테이블에서 최근 7일 활성 사용자를 일별 집계하는 SQL 쿼리 작성해줘" — 맥락 + 구체적 목표 + 제약조건.
•결과 검증 방법론
테스트 코드로 자동 검증 / 다른 AI에게 교차 검증 / 결과를 설명시키고 논리 확인 / 작은 단위로 점진적 확인.

File Structure
•설정 파일 구조

  settings.json      ← 전역 설정

  skills/

Prompt Patterns
•복사해서 바로 쓰는 프롬프트
// 분석
"이 데이터의 패턴과 이상치를 찾아줘"
"이 코드의 성능 병목점을 분석해줘"

// 생성
"이 요구사항으로 HTML 대시보드 만들어줘"
"이 회의록으로 임원 보고서 만들어줘"

// 변환
"이 엑셀 데이터를 시각화해줘"
"이 PDF를 마크다운으로 변환해줘"

// 자동화
"이 작업을 반복하는 Skill 만들어줘"

// 리서치
"경쟁사 3곳의 전략을 비교 분석해줘"
