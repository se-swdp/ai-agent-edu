# KBO 데모 데이터 패키지

> **목적**: 일반 사무업무에서 AI(Claude Code)가 다목적 인터페이스로 작동하는 것을 KBO 야구 데이터로 시연.
> **수집일**: 2026-04-13
> **상태**: 2026 시즌 진행 중 (시즌 초반 13경기 기준), 2024 시즌 완전 데이터

## 데모 추천 흐름

1. **시나리오 1 (3분)** — `team_standings_2024.csv` 활용
   - 지저분한 raw 데이터 정제 → 깔끔한 표
   - 사무업무 매핑: 영업 raw → 보고용 표

2. **시나리오 2 (4분)** — `hitters_2026_top30.csv` 활용
   - 피벗·차트 자동 생성
   - 사무업무 매핑: 월별 KPI 시각화

3. **시나리오 3 (5분)** — `team_standings_2026.csv` + `hitters_2026_top30.csv` 결합
   - 인터랙티브 HTML 대시보드 5분 만에
   - 사무업무 매핑: 경영 현황판

4. **시나리오 4 (5분)** — `pitchers_2026_top15.csv` 활용
   - 이상치·상관관계 탐지
   - 사무업무 매핑: 실적 이상 거래처 탐지

5. **시나리오 5 (5분)** — 모든 데이터 결합
   - 임원 보고서 자동 작성 (Markdown → PDF)
   - 사무업무 매핑: 월간 경영 분석 리포트

각 시나리오의 정확한 프롬프트와 시연 흐름은 `scenarios.md` 참조.

## Q&A 예상 답변

**Q. 데이터가 진짜인가?**
A. 네, KBO 공식 사이트 (koreabaseball.com)에서 직접 가져온 데이터입니다.

**Q. 사내 데이터로도 할 수 있나?**
A. 동일하게 가능합니다. 사외망에서는 익명화·가상 데이터로 시연 후, 사내망 환경에서 실제 데이터 적용.

**Q. 프롬프트 작성이 어렵지 않나?**
A. 시나리오 1 수준은 누구나 가능. 시나리오 4-5는 1-2주 학습으로 도달.

**Q. 결과가 틀리면?**
A. AI 출력은 항상 검증 단계를 거쳐야 합니다 — 시나리오 4에 "교차 검증" 포함.
