# solution-dev-seminar — HANDOFF

Solution 개발실 대상 Claude Code BP 세미나 (방송형, 1시간~1시간 30분, 실습 없음).
이 문서는 **사내망에서 내용을 채워 완성**하기 위한 가이드다. 발표 골격·카피·디자인은 완성 상태이고,
`FILL-IN` 표시가 있는 6개 슬라이드만 실제 사내 데이터로 교체하면 된다.

## 덱 개요

- 파일: `presentations/solution-dev-seminar/index.html` (단일 파일, 51슬라이드)
- 디자인: all-hands-seminar(2026.7) 시스템 그대로 — 스케일 207.5%, 카드 시스템, 다크 고정
- partMap: Opening(0-2) / 격차의 정체(3-9) / 도구의 지도(10-21) / 업무에 녹이기(22-33) /
  잘 쓰는 사람의 습관(34-42) / 개인에서 팀으로(43-47) / Closing(48-50)
  — **슬라이드 추가·삭제 시 하단 `partMap` 배열을 반드시 같이 갱신**
- 등록: `presentations/index.html` 10번 카드 (Dev)

## 시간 배분 (90분 기준)

| 구간 | 슬라이드 | 시간 |
|---|---|---|
| Opening + ① 격차의 정체 | 0–9 | ~12분 |
| ② 도구의 지도 | 10–21 | ~20분 |
| ③ 업무에 녹이기 (BP 코어 + 데모) | 22–33 | ~30분 |
| ④ 잘 쓰는 사람의 습관 | 34–42 | ~15분 |
| ⑤ 개인에서 팀으로 + Closing | 43–50 | ~8분 |
| Q&A | 50 | ~5분+ |

60분으로 줄일 때: ①에서 4·5번 중 1장, ②에서 16(모델 티어)·18(Rules·Memory) 통합, ④에서 38(단일 숫자) 스킵.

## FILL-IN 슬라이드 6장 — 채움 가이드

각 슬라이드에는 보라색 점선 박스(채움 가이드)와 "사내 에이전트에게" 프롬프트가 이미 들어 있다.
채운 뒤에는 **점선 박스(.fill-wrap)를 삭제**하고, 예시 콘텐츠의 `sample-dim` 클래스를 제거하면 된다.

### #24 내 작업 환경 — 통합 지도
- 실제 연동 상태(시스템명·MCP/스킬명·오가는 데이터)로 표 교체. 다이어그램 이미지로 승격 권장.
- 이미지 경로는 반드시 절대경로: `/presentations/assets/images/...` (Firebase cleanUrls 때문에 상대경로 깨짐)
- 프롬프트: 내 `~/.claude` 설정·MCP 구성·스킬 목록을 읽고, Claude Code 중심 아키텍처 한 장으로 정리.

### #25 연동 인벤토리
- 대표 6~8개만 (청중이 따라 만들 수 있는 것 위주). 각 행에 "줄여준 것" 한 줄 추가 시 설득력 상승.
- 프롬프트: 스킬 디렉토리와 `.mcp.json`을 읽고 표 생성 — 분류/이름/하는 일/줄여준 것.

### #27 BP 1 — 티켓에서 PR까지
- 실제 케이스 하나를 처음부터 끝까지: 티켓 번호·실제 지시문·소요 시간.
- **사람이 개입한 지점을 반드시 표시** — 전자동처럼 보이면 신뢰 하락.
- 스크린샷은 티켓·PR 각 1장. diff 전체 불필요.

### #28 BP 2 — 문서에서 스펙으로
- WIKI를 컨텍스트로 쓴 사례 + (있다면) 작업 결과 → 문서 자동 갱신 사례. 후자가 더 강한 BP.

### #29 BP 3 — 운영 속으로 (SWDP)
- 실제 운용 중인 것만: 모니터링·빌드 실패 분석·배포 점검 등.
- 핵심 인상: "사람이 안 보는 시간에 무엇을 해두는가".

### #30 DEMO 영상
- 3~5분, 티켓→PR 흐름 추천. 실패→복구 장면 하나 포함 시 오히려 신뢰 상승.
- 영상: `/presentations/assets/videos/solution-dev-demo.mp4` (절대경로).
  ffmpeg 재인코딩 필수: `-c:v libx264 -crf 25 -preset veryfast -pix_fmt yuv420p -movflags +faststart`
- 슬라이드 안 주석 처리된 `<video>` 태그 활성화 + 안내 박스 삭제.

### #31 효과 측정
- 측정 가능한 수치만, 보수적으로. **수치가 없으면 이 슬라이드는 삭제** (partMap 갱신 잊지 말 것).

### #47 우리 개발실 제안
- 실행 가능한 제안 2~3개. 발표자가 오너십 잡을 1개 포함 시 가장 강함.
