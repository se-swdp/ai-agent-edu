## 1. 전역 톤 & 디자인 규칙
- **카피 = 명사(체언) 종결** 또는 차분한 `~다` 단정형. **금지:** `~합니다/입니다/됩니다/합시다/봅시다/있습니다`.
- **담담한 톤** — 오버한 헤드라인 금지 (예: "세상은 이미 움직였다" → "이미 일어난 변화").
- **뒷자리 가독성** — 글씨 충분히 크게.
- **정렬/통일성/완결성** — 표·카드의 행 높이·열 정렬 일관. arch-layer는 표처럼 정렬(역할 왼쪽·브랜드 같은 x·설명 오른쪽). comp 카드 라벨 상단 정렬, 카드별 줄 수 통일. 설명열(nowrap)은 짧게.

## 3. Part I 구성 (맥락)
1. **도입 = 업계 기준선**: 세상이 어떻게 돌아가는가 · PM:개발자 비율 역전(1:7→2:1, Andrew Ng) · 개발 업무 전반 확산.
2. **업계 기준선**: 채택률 등 지표.
3. **챗봇 ≠ 에이전트 → 팀(스웜)**: 차이 + 팀(스웜)으로 발전 + 컴퓨팅(토큰)으로 품질 사는 영역. **현재가 팀(스웜) 단계**임을 명시.
4. **토큰 = 성능 시대**: 모델 발전은 더뎌짐 / 에이전트 능력↑ → 산출물 품질↑.
5. **비유로 이해하는 구조 + 성능 공식 필수**. 에이전트→팀 맥락 연결.
- 자동완성→대화→에이전트→팀 흐름: **축소(한 줄)**, 구두 설명. "현재 여기" 배지 제거(색으로 표현).
- **SaaSpocalypse 제거.**
- "챗봇 사용자는 에이전트 생산성 가늠 못함" → "**애초에 할 수 있는 일이 다르다**" (올드한 어투 교체).
- 이미 일어난 변화 — GitHub README 사람/에이전트 분기.
- **AI 인프라 병목 = 부록(여담), 간추림** (메모리 계층 상세 슬라이드 제거). 왜 메모리 수요 폭증.
- **역할의 변화 = 핵심 인사이트.**
- **인지 노동의 심화 = 주요 → 색·크기 강조.**
- **AI 만능론 hype-curve 이미지 필수.**
- 생산성의 역설과 도파민 루프.
- S/W 개발팀 현재 진행 중인 사항.
- **AI는 증폭기** — 쓴다고 무조건 좋아지지 않음, 인지 부하·실수 필터링 훈련 필요.

## 4. Part II 구성 (Claude Code Hands-on)
- **환경 셋업**: 터미널(PowerShell/cmd OK, 기왕이면 WezTerm+bash 기본 추천) · alias `claude --dangerously-skip-permissions` · **statusline 먼저 설정**(현재 세팅 기반 재현 프롬프트) · 모든 설정은 Claude에게 시킴 · **"내가 할 것 / Claude가 할 것 구분해서 알려줘"가 초기 핵심**.

- **Skill**: 설명 + 만들기 전에 찾아라. **스킬세트는 시즌마다 유행 바뀜 — superpowers/SuperClaude/oh-my-claude-code 중 인기/맞는 것 택1** (고정 추천 X). 주기적으로 인기 plugin/skill 조사해 적용.
- **Lab**: 자유 주제 리포트 작성 → skill화 → 다른 주제로 동일 포맷 재현 확인.
