한 줄로

질문이 아니라 — 그다음이 가른다

초심자는 묻고 받아쓴다. 숙련자는 묻되 — 가설을 얹고, 첫 답을 의심하고, 완료를 심문한다.

오늘의 8가지

  • ① AI를 비판적 동료로 세팅  ·  ② 가설을 던진다
  • ③ 첫 답은 초안, 표면을 뚫는다  ·  ④ 완료를 심문한다
  • ⑤ 크게 맡기되 끝은 쥔다  ·  ⑥ 도구를 다시 만든다
  • ⑦ 정성을 차등 배분한다  ·  ⑧ 멈출 줄 안다

세팅 → 위임 → 검증 → 운영 — 무게중심은 검증이다

① AI의 체급을 먼저 정해라

그냥 질문

"최신 기술 트렌드 알려줘"

프로토콜 부여

"너는 깐깐한 수석 검토자다 — 칭찬은 빼고, 내 논리적 맹점을 근거와 함께 즉시 반박해라"

AI를 부하가 아니라 — 나보다 꼼꼼한 '깐깐한 사수'로 세팅한다

① 매 턴 주입 대신, 한 번 박아둔다

채팅

대화마다 페르소나·원칙을 다시 주입

코드

CLAUDE.md · 에이전트 · 룰에 영구 코드화 — 한 번 정하면 매번 적용

AI의 체급은 — 내가 깔아준 프로토콜이 결정한다

② 빈 질문이 아니라, 가설을 얹는다

빈 명령

"이 모듈 리팩토링해줘"

가설을 얹은 위임

"결제 모듈이 중복 로직 때문에 느린 것 같거든 — 원인 짚고 리팩토링한 다음, 실제로 돌려서 빨라졌는지 보여줘"

[현재 상태] + [내 가설] + [제약] — 채워야 할 빈칸이 있는 브리핑

② 도메인이 달라도, 모양은 같다

  • 개발 — "마이그레이션이 뭔가 조용히 떨군 것 같거든, 베이스라인이랑 비교해줘"
  • 인프라 — "토큰 비용이 너무 들어서, 내부망 모델을 붙여쓰는거야"
  • 거실 매트 — "두 개 붙은 하나짜리야, 어떤 식이 좋을지"

사람한테 일 넘기듯 — 중간에 "이해했지?"로 맥락을 맞춘다

③ 첫 답은 결과물이 아니라 초안이다

  • 되묻는다 — "방금 답에서 가장 약한 가정이 뭐야?"
  • 반대시킨다 — "이 결론을 가장 세게 반박하면?"
  • 구체를 강제한다 — "일반론 말고, 숫자와 사례로"

표면적인 답이 나올 자리엔 미리 못을 박는다 — "뻔한거 말고"

"실제로 돌려봤어?"

④ "다 했습니다"라는 말 자체를 안 믿는다 — 증거를 요구한다

빌드 출력 · 통과 테스트 수 · 스크린샷 · 실제 데이터

④ 단일 숫자에 속지 마라

"line이야 function이야? TC가 몇 개고 어떻게 구성돼 있어?"

단일 숫자는 claim이고 — TC 개수 × 분기 구성이 evidence다

⑤ 스텝이 아니라 스코프를 위임한다

마이크로매니징

"먼저 이 파일 열고, 그다음…" — AI가 그냥 느린 손이 된다

스코프 위임

"이 세 화면 모바일까지, 단 외부 라이브러리는 추가 금지" — 경계가 분명하다

방법은 AI가 더 잘 정한다 / 방향(무엇을·어디까지·무엇은 절대 안 됨)은 끝까지 내가 쥔다

⑥ 도구를 쓰지 말고, 다시 만든다

고비용 작업고성능 모델→ 외부 API
대량·경량 작업내부망 모델→ 사내 게이트웨이

비용이 마음에 안 들면 — 밑단을 바꾼다. 스킬·게이트웨이를 직접 짜서 작업별로 분기

⑦ 정성을 차등 배분한다

⑧ 자율은, 멈출 줄 알아야 책임이다

  • 정지 조건을 명령 출력으로 채점 — "좋아질 때까지"는 반드시 후퇴한다
  • 매 턴 점수표를 출력 — 안 보이는 자율은 자율이 아니라 도박
  • 멈춤은 외과수술로 — 이름이 아니라 포트로

켜는 건 한 줄 — 어려운 건 멈출 줄 아는 것이다

이건 내 느낌이 아니다

서로 다른 AI 여섯이 내 대화를 분석했고

같은 결론에 수렴했다 — 위 8가지는 기법이 아니라 관찰된 행동이다

한 줄로 묶으면

AI를 답변기가 아니라 — 비판적 동료로 설계해서 부린다

가설을 던지고, 증거와 숫자에 닿을 때까지 심문하고, 재사용 가능한 산출물로 닫는다

끝은 — "증거 보여줘"로 닫아라

부려 보면, 안다

읽어서가 아니라 — 직접 위임하고 의심하고 다시 시키면서 몸에 붙는다
